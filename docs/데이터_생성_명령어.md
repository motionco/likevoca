## 🔧 매 배치마다 실행할 검증 명령어

### 기본 검증 (매 배치마다):
```bash
# 1. 중복 검사 및 기본 검증
python validate.py

# 2. 데이터 누적 처리
python accumulator.py

# 3. 현재 상태 확인
python stats.py
```

### 강화 검증 (매 5배치마다):
```bash
# 4. 중복 검사 강화
python validate.py --strict

# 5. 트랜잭션 로그 확인
cat data_tracking_log.json | jq '.summary'

# 6. CSV 파일 무결성 검증
head -1 concepts_template_add.csv | wc -w
head -1 examples_template_add.csv | wc -w
head -1 grammar_template_add.csv | wc -w
```

### 백업 (매 10배치마다):
```bash
# 7. 전체 백업 생성
python backup.py

# 8. 백업 폴더 확인
ls -la backups/ | tail -5
```

## 📊 진행 상황 모니터링 명령어

### 실시간 통계:
```bash
# 현재까지 생성된 데이터 통계
python stats.py --summary

# 도메인별 분포 확인
python stats.py --domain

# 난이도별 분포 확인
python stats.py --difficulty

# 목적별 분포 확인
python stats.py --purpose
```

### 품질 검증:
```bash
# 필드 완성도 검사
python validate.py --completeness

# 언어 일관성 검사
python validate.py --language-consistency

# 예문 중복 검사
python validate.py --example-duplicates
```

### 오류 처리:
```bash
# 오류 발생 시 마지막 성공 지점으로 복원
python restore.py --last-successful

# 특정 시점으로 복원
python restore.py --timestamp "2025-08-09_14-30-00"

# 손상된 CSV 파일 복구
python repair.py --file concepts_template_add.csv
```

## ⚠️ 주의사항

### 명령어 실행 순서:
1. **생성**: `python generate.py`
2. **검증**: `python validate.py`
3. **누적**: `python accumulator.py`
4. **백업**: `python backup.py` (정기적)

### 오류 발생 시:
1. **즉시 중단**: 오류 발생 시 다음 배치로 진행하지 말고 문제 해결
2. **로그 확인**: `data_tracking_log.json`에서 오류 원인 파악
3. **복원 실행**: 필요시 이전 백업으로 복원 후 재시작

### 데이터 품질 기준:
- **필드 완성도**: 95% 이상
- **언어 일관성**: 5개 언어 모두 의미 일치
- **concept_id 고유성**: 100% 보장
- **예문 차별화**: concepts/examples/grammar 간 예문 상이

이 명령어 가이드를 따라 체계적으로 10,000개의 고품질 데이터를 생성할 수 있습니다.
