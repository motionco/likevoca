# 데이터 업로드 방법 안내

## 📋 개요

이 가이드는 확장된 다국어 학습 시스템에 새로운 개념 데이터를 업로드하는 방법을 설명합니다. CSV와 JSON 두 가지 형식을 지원하며, 각각의 장단점과 사용 방법을 안내합니다.

## 🚀 빠른 시작

### 1단계: 업로드 모달 열기

- 메인 페이지에서 **"대량 개념 추가"** 버튼 클릭
- 또는 개념 목록 페이지에서 **"+"** 버튼 → **"대량 추가"** 선택

### 2단계: 파일 형식 선택

- **CSV**: 스프레드시트(Excel, Google Sheets)에서 작업하기 편리
- **JSON**: 복잡한 구조와 모든 기능을 완전히 지원

### 3단계: 템플릿 다운로드

- **"템플릿 다운로드"** 버튼 클릭
- 선택한 형식에 맞는 샘플 파일 다운로드

### 4단계: 데이터 입력

- 다운로드한 템플릿에 데이터 입력
- [데이터 구조 가이드](#데이터-구조-설명) 참조

### 5단계: 파일 업로드

- 완성된 파일을 드래그 앤 드롭하거나 **"파일 선택"** 클릭
- **"가져오기 시작"** 버튼 클릭

## 📊 CSV 형식 사용법

### 장점

- Excel, Google Sheets 등에서 쉽게 편집 가능
- 대량 데이터 입력에 적합
- 비개발자도 쉽게 사용 가능

### 제한사항

- 복잡한 구조의 일부 기능 제한
- 중첩된 데이터 표현의 제약

### CSV 필수 컬럼

| 컬럼명        | 설명          | 필수 여부 | 예시                     |
| ------------- | ------------- | --------- | ------------------------ |
| domain        | 개념 도메인   | 필수      | food, daily, nature      |
| category      | 개념 카테고리 | 필수      | fruit, greeting, weather |
| unicode_emoji | 대표 이모지   | 선택      | 🍎, 👋, ☀️               |
| korean_word   | 한국어 단어   | 필수      | 사과                     |
| english_word  | 영어 단어     | 필수      | apple                    |
| japanese_word | 일본어 단어   | 선택      | りんご                   |
| chinese_word  | 중국어 단어   | 선택      | 苹果                     |

### CSV 데이터 입력 팁

#### 1. 다중 값 입력 (파이프 구분)

```csv
korean_synonyms,korean_word_family
"안녕|반갑습니다","안녕|안녕히|안녕하다"
```

#### 2. 빈도 정보가 있는 연어 (콜론 구분)

```csv
korean_collocations
"사과를 먹다:high|빨간 사과:high|사과 한 개:medium"
```

#### 3. 예문 입력

```csv
example_1_korean,example_1_english,example_1_context,example_1_priority
"아침에 사과를 먹어요.","I eat an apple in the morning.","daily_routine","high"
```

## 🔧 JSON 형식 사용법

### 장점

- 모든 확장 기능 완전 지원
- 복잡한 중첩 구조 표현 가능
- 개발자 친화적

### 제한사항

- JSON 문법 지식 필요
- 수동 편집이 상대적으로 복잡

### JSON 기본 구조

```json
[
  {
    "concept_info": {
      "domain": "food",
      "category": "fruit",
      "difficulty": "basic",
      "tags": ["everyday", "healthy"],
      "unicode_emoji": "🍎",
      "color_theme": "#FF6B6B"
    },
    "expressions": {
      "korean": {
        "word": "사과",
        "pronunciation": "sa-gwa",
        "definition": "둥글고 단맛이 나는 열매"
      }
    }
  }
]
```

## 📖 데이터 구조 설명

### 개념 정보 (concept_info)

- **domain**: 개념이 속하는 큰 분야 (예: food, daily, nature)
- **category**: 세부 카테고리 (예: fruit, greeting, weather)
- **difficulty**: 난이도 (basic, intermediate, advanced)
- **tags**: 태그 배열 (검색 및 분류용)
- **unicode_emoji**: 대표 이모지
- **color_theme**: 테마 색상 (헥스 코드)
- **quiz_frequency**: 퀴즈 출현 빈도 (low, medium, high, very_high)
- **game_types**: 지원 게임 타입 배열

### 언어별 표현 (expressions)

각 언어(korean, english, japanese, chinese)별로:

- **word**: 단어 (필수)
- **pronunciation**: 발음 표기
- **definition**: 정의
- **part_of_speech**: 품사
- **level**: 난이도 레벨
- **synonyms**: 동의어 배열
- **antonyms**: 반의어 배열
- **word_family**: 어족 배열
- **collocations**: 연어 배열 (빈도 정보 포함)

### 예문 (featured_examples)

- **level**: 예문 난이도
- **context**: 사용 맥락
- **priority**: 우선순위
- **translations**: 각 언어별 번역과 문법 노트

### 퀴즈 데이터 (quiz_data)

- **question_types**: 지원 문제 유형
- **difficulty_multiplier**: 난이도 배수
- **common_mistakes**: 흔한 실수 패턴
- **hint_text**: 언어별 힌트 텍스트

### 게임 데이터 (game_data)

- **memory_card**: 메모리 게임용 카드 정보
- **word_puzzle**: 단어 퍼즐 데이터
- **pronunciation_game**: 발음 게임 데이터

### 학습 메타데이터 (learning_metadata)

- **memorization_difficulty**: 암기 난이도 (1-5)
- **pronunciation_difficulty**: 발음 난이도 (1-5)
- **usage_frequency**: 사용 빈도
- **cultural_importance**: 문화적 중요도

## ⚠️ 주의사항 및 팁

### 1. 데이터 검증

- 업로드 전 필수 필드가 모두 채워졌는지 확인
- 언어별 단어가 정확한지 검토
- 이모지가 올바르게 표시되는지 확인

### 2. 파일 크기 제한

- 한 번에 최대 1000개 개념까지 업로드 권장
- 대용량 파일은 여러 번에 나누어 업로드

### 3. 중복 방지

- 동일한 단어 조합의 개념이 이미 있는지 확인
- 기존 데이터와 중복될 경우 업로드 시 오류 발생

### 4. 인코딩 문제 해결

- CSV 파일은 UTF-8 인코딩으로 저장
- Excel에서 저장 시 "CSV UTF-8" 형식 선택

### 5. 특수 문자 처리

- CSV에서 쉼표(,)나 따옴표(")가 포함된 텍스트는 큰따옴표로 감싸기
- JSON에서 백슬래시(\)나 따옴표는 이스케이프 처리

## 🛠️ 문제 해결

### 업로드 실패 시

1. 파일 형식이 올바른지 확인 (.csv 또는 .json)
2. 필수 필드가 누락되지 않았는지 점검
3. 특수 문자나 인코딩 문제 확인
4. 브라우저 콘솔에서 오류 메시지 확인

### 데이터 누락 시

1. 템플릿과 입력 데이터 구조 비교
2. JSON 문법 검증기로 유효성 확인
3. 샘플 데이터와 비교하여 형식 점검

### 성능 최적화

1. 배치 크기를 적절히 조절 (100-500개 권장)
2. 네트워크 상태가 안정적일 때 업로드
3. 대용량 미디어 파일은 URL 링크 사용
